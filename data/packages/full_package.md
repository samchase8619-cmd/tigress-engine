### FULL CORE SYSTEMS

**RPCM_MASTER_MODEL_V7 — Recursive Path-Dependent Constraint Model**

```
Core loop:  C → I → B → F → C'

C  = Constraints
I  = Incentives
B  = Behavior
F  = Feedback
C' = New constraint state (path-dependent — history matters)

Key parameters:
  alpha  = constraint sensitivity
  beta   = incentive weight
  gamma  = feedback decay rate
  K      = system capacity
  P      = path-dependence coefficient

Rule: past states narrow future options. History is not neutral.
```

**INSTABILITY_ENGINE_V2**

```
States:
  stable / strained / overloaded / unstable / fracture / collapse

Duration bands:
  How long a state persists before it forces a transition.

Recovery bands:
  Minimum conditions required to return to stable.

Governance variable:
  Who or what controls rebalancing. If absent → fracture accelerates.

Degradation requirement:
  Instability ≠ disruption. Degradation must be present.
```

**CONVERSION_EFFICIENCY_V1**

```
Formula: useful_output / total_input

Interpretation:
  Low efficiency alone → inefficiency, not instability
  Low efficiency + sustained degradation → instability candidate
  Collapsing efficiency + no governance → fracture risk
```

**SYSTEM_CAP_RULE_V1**

```
Every system has a maximum absorption rate.
If pressure exceeds absorption rate for long enough → state degrades.
Cap is not fixed — it can be expanded (more capacity) or compressed (more constraint).
```

**DISTRIBUTION_LAYER_V1**

```
Segmented analysis: same pressure → different outcomes by group
K-shaped divergence:
  Top segment: recovers or gains
  Bottom segment: degrades or loses
  Middle: narrows

Risk: divergence masks aggregate stability metrics.
      System "looks fine" until lower segment breaks.
```

**K_DIVERGENCE_RULE_V1**

```
When distribution becomes K-shaped:
  Aggregate metrics are misleading.
  Stability at the top does not mean stability at the system level.
  Bottom segment fracture propagates upward over time.
```

**SIGNAL_LAYER_V1**

```
Chain: Reality → Data → Media/Filter → Perception

Signal divergence = gap between reality and perception

Causes:
  - data lag
  - filter distortion (media, authority, ideology)
  - incentive-driven suppression

Risk: decisions made on false signal state worsen underlying problem.
```

**SIGNAL_DIVERGENCE_RULE_V1**

```
IF signal_divergence is high:
  classification accuracy drops
  intervention timing fails
  false stability becomes likely
```

**SLEEPWALKER_DYNAMICS_V1**

```
Pattern:
  Locally rational actors + low coordination + distorted signals
  → catastrophic emergent outcome

No single actor intends the outcome.
Each actor is responding rationally to their local signal.
Aggregate behavior is irrational at the system level.

Classic cases: financial crises, arms races, institutional collapse
```

**LOCAL_RATIONALITY_FAILURE_V1**

```
Individual rationality ≠ collective rationality.
When coordination is low and signals are distorted:
  Local optimization produces systemic degradation.
```

**BREAK_DETECTION_ENGINE_V1**

```
States:
  stable → strained → rebalancing → fracturing → broken

Costly defection rule:
  If defecting costs more than staying aligned, system holds.

Costly alignment rule:
  If staying aligned costs more than defecting, system fractures.

First break types:
  CONTROL      — can't direct outcomes
  INFORMATION  — bad or missing data
  COORDINATION — actors misaligned
  TIME         — can't keep pace
```

| Adaptation capacity    | Intact / reduced / depleted                |

**VALIDATION_LOG_V1**

```
Prediction format:
  IF [condition] THEN [outcome] WITHIN [timeframe/trigger]

Validation steps:
  1. Make prediction before observing outcome
  2. Observe outcome
  3. Compare: match / partial / failed
  4. Log result
  5. Patch if boundary was wrong

Falsification rule:
  A prediction that cannot be wrong is not a prediction.
```

**GENERIC_INTERVENTION_LAYER_V1**

```
Logic:
  prediction → identify first failing function → intervene before fracture

NOT: treat symptoms
BUT: reduce constraint OR increase capacity OR repair signal

Priority order:
  1. Restore signal alignment (can't fix what you can't see)
  2. Reduce constraint load (buy time)
  3. Increase adaptation capacity (structural fix)
  4. Restore coordination (prevent cascade)
```

**Boundary case method:**
```
Test the edge of each state transition.
What is the minimum change that shifts state?
That is the boundary.
If boundary is wrong → patch.
```

**Adaptive disruption test:**
```
Introduce a disruption.
Does the system recover? How fast? What was the cost?
Recovery = adaptation.
No recovery + ongoing degradation = instability.
```

**Inefficiency vs instability test:**
```
Is output low? Yes.
Is degradation present? No → inefficiency, not instability.
Is degradation present? Yes → instability candidate.
Is function being lost? Yes → instability confirmed.
```

**Hidden degradation test:**
```
Surface metrics look stable.
Check: who is compensating?
Check: what is being deferred?
Check: what would happen if the compensator stopped?
If answer = collapse → false stability.
```

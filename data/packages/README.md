# TigressOS — Package Index

Three copy-paste packages. Pick one based on what you need.

---

## Packages

| File                | Purpose                                         | When to use                                  |
|---------------------|-------------------------------------------------|----------------------------------------------|
| `model_package.md`  | Portable analysis engine for any AI             | Sharing the model without the OS             |
| `mini_package.md`   | Fast TigressOS reload                           | Continuing work in a new session quickly     |
| `full_package.md`   | Complete TigressOS restoration                  | Rebuilding from scratch / deep session       |

---

## Load Order

**For another AI (no TigressOS):**
```
1. model_package.md only
```

**For fast TigressOS reload:**
```
1. mini_package.md
```

**For full TigressOS restoration:**
```
1. full_package.md
```

---

## What each package contains

### model_package.md
- Core variables and rule
- State ladder
- First break types
- Key distinctions (adaptation vs instability, false recovery, false stability)
- Human layer
- Case log format + example
- Parameter tracking table
- How to run a case
- Intervention logic

### mini_package.md
Everything in model_package, plus:
- TigressOS identity and boot priorities
- Command reference
- RPCM loop
- Validation system
- Casework methods
- World / game anchors (archetypes, petals, class structure, cards)
- Recovery anchors for active cases and world/game state

### full_package.md
Everything in mini_package, plus:
- Full module definitions for all 13 core model systems
- Full continuity systems (branch save / collapse / reconciliation)
- Full creative systems (world gradient, node templates, card engine, D&D overlay)
- Full library stack (6 libraries)
- Full harddrive (model, case, paper, OS, world, game, symbolic archives)
- Complete case archive with logged states

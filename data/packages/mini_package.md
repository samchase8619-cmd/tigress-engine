## BLOCK 3 — MINI LIBRARIES

### CORE MODEL (MINI)

**RPCM — Recursive Path-Dependent Constraint Model**
```
C → I → B → F → C'

C  = Constraints (what limits behavior)
I  = Incentives (what drives decisions)
B  = Behavior (observable actions)
F  = Feedback (what loops back)
C' = New constraint state

Key parameters: alpha / beta / gamma / K / P
```

**INSTABILITY ENGINE**
```
States: stable / strained / overloaded / unstable / fracture / collapse
- Duration bands: how long a state persists before shifting
- Recovery bands: what's needed to return to stable
- Governance variable: who or what controls rebalancing
- Degradation required: instability ≠ disruption alone
```

**BREAK DETECTION**
```
stable → strained → rebalancing → fracturing → broken

Costly defection rule:  if defecting costs more than staying, system holds
Costly alignment rule:  if alignment costs more than defecting, system fractures
```

**PARAMETER TRACKING**

| Parameter              | What to track                              |
|------------------------|--------------------------------------------|
| Constraint load        | High / medium / low                        |
| Adaptation capacity    | Intact / reduced / depleted                |
| Signal alignment       | Accurate / lagging / distorted             |
| Coordination           | Functioning / degraded / broken            |
| Distribution           | Even / K-shaped                            |
| Conversion efficiency  | High / low / collapsing                    |
| Intra-group stability  | Cohesive / fragmenting                     |
| Cross-group alignment  | Aligned / misaligned                       |

**SIGNAL LAYER**
```
Reality → Data → Media/Filter → Perception

Signal divergence: when perception no longer matches reality
Risk: decisions made on false signal state
```

**DISTRIBUTION LAYER**
```
Segmented analysis: who gains, who loses, from the same pressure
K-shaped divergence: top segment recovers, bottom segment degrades
```

**CONVERSION EFFICIENCY**
```
useful_output / total_input

Low efficiency ≠ instability
Low efficiency + degradation = instability candidate
```

### VALIDATION (MINI)

```
Validation loop:
  predict → observe → compare → log → patch if wrong

Falsification rule:
  A prediction must be able to be wrong to be useful.

Failure classes:
  - boundary failure (wrong threshold)
  - classification failure (wrong state label)
  - driver failure (wrong cause identified)

Threshold testing:
  What's the minimum change needed to flip the state?
```
